case class Rack(id: String, description: Option[String], project: Option[String], tags: List[ComponentTag], locationID: Option[String], contentID: Option[String], layout: Division) extends Component with Location with Container with Transferrable with JiraProject with Product with Serializable

Rack - it's a container (can have initial contents) and can have a location (e.g., a freezer) and can have stuff transferred out of it. Normally a rack has tubes that are subcomponents but for efficiency reasons, if all the tubes of a rack have the same contents and are always used together then the rack itself becomes the container where everything that happens to it happens to all the tubes on the rack.
